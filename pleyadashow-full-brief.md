## 1. Технічна частина

**Тип сайту:** статичний лендинг (HTML/CSS/JS), одна сторінка, без важких фреймворків
**Мова сайту:** українська
**Інструмент верстки:** Claude Code + skill ui-ux-pro-max (опційно)
**Хостинг:** GitHub + GitHub Pages
- Репозиторій: `Pleyadashow/pleyadasite` (Public)
- Локальний шлях: `C:\Projects\pleyadasite`
- Pages: Deploy from branch `main`, root

**Домен:** `animatory.in.ua` (в процесі реєстрації)
**Подальші правки:** через Claude Code (команди на кшталт "додай фото") → git commit + push → сайт оновлюється сам. Дрібні заміни фото можна робити і без агента — напряму через GitHub веб-інтерфейс.

### SEO-фундамент
- Мета-теги (title, description) українською під кожну секцію
- Open Graph теги
- Alt-теги для всіх фото — українською
- sitemap.xml, robots.txt
- Schema.org: LocalBusiness (Коломия + Івано-Франківська область), Review для відгуків
- Оптимізація швидкості (стиснення фото, лінива підвантаження галереї)
- Семантичний HTML, коректна ієрархія заголовків
- Заготовка під Google Analytics / Search Console

### GEO/AEO-фундамент (видимість в AI-пошуку)
- Конкретні, розмовні формулювання замість маркетингового жаргону
- FAQ-блок з розміткою Schema.org FAQPage
- Чіткі заголовки-питання з прямою відповіддю під ними

### Адаптивність
- Mobile-first, коректна робота на смартфонах, планшетах, десктопах
- Пріоритет — мобільна версія (основна аудиторія заходить з телефону)

### Контакти / канали зв'язку
**Блок "Написати нам"** (перед FAQ/контактами):
- Текст-інструкція: вказати дату заходу, місце, орієнтовну кількість дітей, вік іменинника

- Телефон НЕ вказувати

**Блок "Підписуйтесь"** (окремий, нижче/в кінці галереї):
- Instagram — `instagram.com/pleyadashow`
- Telegram — `t.me/Fetyushka`

## 2. Бренд і позиціонування

- **Назва на сайті:** Pleyadashow (текстовий логотип, без іконки)
- **Позиціонування:** аніматори та ростові ляльки по всій Івано-Франківській області
- **Аудиторія:** батьки дітей 3–12 років, гендерно-нейтральний дизайн
- **Тон:** преміальність, стриманість — не "дитячо-конфетний", не "підлітковий", не "дешево і весело"

### Вайб "зірковості" (непряма подача)
- Команда часто зайнята — відповідають не миттєво (саме зараз проводять чиєсь свято)
- В Instagram відповідають швидше
- Клієнти часто приходять за рекомендацією
- Одним стриманим реченням біля контактів: "через високий попит дати розписані наперед, пишіть заздалегідь"
- **НЕ використовувати:** "ми найкращі", "у нас черга", терміновий CTA на кшталт "встигніть забронювати!!"

## 4.1. Принцип GEO/SEO (без ілюзій щодо гарантій)

Мета — зробити максимум з боку сайту (текст + Schema-розмітка + реальні відгуки з локаціями), розуміючи, що жоден сайт не гарантує потрапляння в конкретну AI-відповідь по конкретному запиту — це залежить від алгоритмів, які поза контролем. Після запуску верстки — окремим кроком налаштувати **Google Business Profile** (безкоштовна картка в Google Maps/Пошуку) із зоною обслуговування аж до конкретних сіл — це часто сильніше впливає на локальні AI-відповіді, ніж сам сайт.

## 5.2. Логіка підбору команди аніматорів

⚠️ **На сайті — БЕЗ конкретних цифр і без таблиці порогів.** Тільки текстовий принцип, без слів на кшталт "хаос" чи будь-якого порівняння з конкурентами — мова тільки про власний підхід:

> Ми не працюємо за шаблоном "один аніматор на будь-яке свято". Команду підбираємо індивідуально під кожну подію — з урахуванням кількості дітей, формату та локації, спираючись на величезний досвід та сотні проведених яскравих подій. Так кожна дитина лишається в грі, а свято — справді яскравою подією.

Конкретні цифри (нижче) — **робочий довідник для власника в особистому спілкуванні**, на сайт НЕ виводити:

| Ситуація | Орієнтовна кількість аніматорів (внутрішньо, не для сайту) |
|---|---|
| Невелика компанія (5-6 дітей) | Буває і 1 — залежить від формату |
| До 15 дітей | Як правило, 2 аніматори |
| 15 дітей і більше | Як правило, 3 аніматори |

Фінальне рішення завжди після брифу від клієнта (дата, кількість дітей, формат, локація), в одній логіці з ціноутворенням (розділ 6).
